# Security policy

## Supported versions

Security fixes are applied to the latest commit on `main`.

## Reporting a vulnerability

Please do not open a public issue for an undisclosed vulnerability. Use
[GitHub private vulnerability reporting](https://github.com/lcarlini/Docker/security/advisories/new)
and include:

- the affected endpoint, image, or configuration;
- steps to reproduce the issue;
- the expected impact;
- any suggested mitigation.

You can expect an initial response within seven days. Once a fix is available,
the advisory can be published with appropriate credit.

## Scope

Reports about secret exposure, container escape paths, unsafe image defaults,
and diagnostic data disclosure are particularly relevant. Automated scanner
output without a reproducible impact may be closed.
