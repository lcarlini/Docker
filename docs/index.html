The idea

Ship the app with its runtime

Docker wraps your .NET app and everything it needs to run into an image. A container is that image running — same behavior on your laptop, CI, or a server.

Source → Dockerfile → Image → Container → HTTP :8080

The flow

How Dockyard builds

The multi-stage Dockerfile builds in layers, then copies only the published output into a small ASP.NET runtime image.

1

Restore dependencies

Start from the .NET SDK image. Copy the .csproj first and run dotnet restore so NuGet packages can be cached when only app code changes.

Dockerfile · restore stage
2

Publish a Release build

Copy the rest of the source and run dotnet publish. That produces the compiled DLL and assets under /app/publish — ready to run, without the SDK.

Dockerfile · build stage
3

Run on a slim image

Switch to mcr.microsoft.com/dotnet/aspnet. Copy only the published files, listen on port 8080, and run as a non-root user. Smaller, safer, closer to production.

Dockerfile · runtime stage
4

Health-check from inside

The image probes itself with dotnet Dockyard.Api.dll --health-check, which hits /health/live. No extra curl package needed.

Dockerfile HEALTHCHECK · Program.cs
5

Compose hardens the runtime

compose.yaml maps the port, sets a read-only root filesystem, drops all capabilities, and adds no-new-privileges. One command: docker compose up --build.

compose.yaml
6

Call the API

Once the container is up, hit diagnostics endpoints like /api/v1/runtime and /api/v1/inspect — useful for testing networking, proxies, and health probes.

http://localhost:8080

Why it matters

Beyond “it builds”

Dockyard shows the habits that keep a .NET container trustworthy once it leaves your machine.

Multi-stage builds

Compile with the SDK; ship only the ASP.NET runtime image.

Least privilege

Non-root user, read-only disk, capabilities dropped.

Health signals

Liveness and readiness endpoints plus an image-level probe.

CI validation

GitHub Actions builds, tests, and verifies the production image.

Try it

One command to sail

From the repository root, build and run the hardened Compose service. 

$ docker compose up --build

$ curl http://localhost:8080/api/v1/runtime
